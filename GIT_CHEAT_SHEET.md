# Git 代码提交操作指南

这份文档记录了本项目日常开发中将代码同步到 GitHub 的标准流程。

---

## 🚀 日常提交三部曲 (最常用)

当你修改了代码（如修改了模型结构、写了新脚本）并测试通过后，请执行以下三个步骤：

### 1. 保存变更 (Add)
将当前文件夹下的所有修改加入“暂存区”。

```bash
git add .
```

### 2. 提交版本 (Commit)
将暂存区的修改打包成一个新版本。
**注意：** 引号内的文字要清晰描述你做了什么（比如 "修复了Dataset的bug" 或 "新增U-Net训练脚本"）。

```bash
git commit -m "这里写你的修改备注"
```

### 3. 推送到云端 (Push)
将本地的新版本上传到 GitHub 仓库。

```bash
git push
```

---

## 🛠 常用辅助命令

### 查看当前状态
如果你不确定哪些文件被改了，或者是否有文件没被追踪，运行：

```bash
git status
```
*   **红色文件：** 修改了但没 add。
*   **绿色文件：** 已经 add 但没 commit。

### 查看提交历史
查看之前的版本记录：

```bash
git log
```
*(按 `q` 键退出查看)*

---

## ⚠️ 注意事项
1.  **大文件限制：** 不要尝试上传超过 100MB 的文件（如 `.pth` 权重或大数据集），`.gitignore` 应该已经配置好忽略它们了。
2.  **先拉后推：** 如果你在 GitHub 网页上修改了代码（比如改了 README），在本地再次提交前，记得先运行 `git pull` 把网页上的修改拉下来，否则会报错。
